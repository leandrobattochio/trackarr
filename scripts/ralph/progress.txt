## Codebase Patterns
- .NET 10 hexagonal arch: Domain (no deps) → Application (refs Domain) → Infrastructure (refs Domain+Application) → Api (refs all three)
- Solution file uses forward-slash paths in src\Project\Project.csproj format in .sln but actual dirs use forward slashes
- docker-compose.yml mounts sqlite_data volume at /data in container; connection string uses /data/trackerstats.db
- appsettings.json holds ConnectionStrings:DefaultConnection and SyncJob:IntervalMinutes
- NOTE: .NET SDK not installed in this environment; build verification requires a .NET 10 SDK
- EF Core 10.0.0 packages: Microsoft.EntityFrameworkCore.Sqlite + Microsoft.EntityFrameworkCore.Design (PrivateAssets=all) in Infrastructure.csproj
- EF Core migrations created manually (no dotnet-ef tooling available); migration timestamp format: YYYYMMDDHHmmss_MigrationName.cs
- AppDbContext lives in Infrastructure/Data/; IntegrationRepository in Infrastructure/Repositories/
- Database.MigrateAsync() called at startup in Program.cs inside a scoped service block
- Plugin pattern: register each ITrackerPlugin as AddSingleton<ITrackerPlugin, Impl>(); registry resolves IEnumerable<ITrackerPlugin> via constructor injection
- ITrackerPluginRegistry is singleton; inject it in controllers/jobs that need to look up plugins by ID or list all
